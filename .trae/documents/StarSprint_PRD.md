# StarSprint 产品需求文档 (PRD)

## 1. 执行摘要

- **问题陈述**：GitHub 每日产生大量新仓库，但用户难以从海量信息中快速发现近期表现最为亮眼、增长速度最快的新兴项目。
- **拟定方案**：StarSprint 是一个轻量级的 React Web 应用，通过 GitHub API 检索指定时间内创建的仓库，并根据 `Stars / 仓库年龄` 计算增长速率，帮助用户识别真正的“增长黑马”。
- **成功标准**：
  - 用户能够设置统计天数、最小 Stars 和返回数量。
  - 应用能准确计算并展示增长速度最快的项目列表。
  - 支持 GitHub Token 提高 API 请求上限。
  - 界面简洁、直观，响应速度快。

## 2. 用户体验与功能需求

- **用户角色**：开发者、技术猎头、开源爱好者。
- **用户故事**：作为一名开发者，我希望通过设置参数（如：过去 7 天创建、Stars > 50），快速找到最近爆火的项目，以便我能第一时间学习或参与其中。
- **验收标准**：
  - 参数输入：天数 (Days)、最小 Stars (Min Stars)、返回数量 (Limit)。
  - 结果展示：表格形式，包含仓库名、描述、Stars、创建日期、增长速度 (Stars/Day)。
  - 仓库链接：点击表格中的仓库名能直接在浏览器中打开 GitHub 页面。
  - Token 支持：用户可以输入 GitHub Personal Access Token (PAT) 以避免 API 频率限制。
- **非目标**：
  - 复杂的仓库趋势图表（MVP 阶段暂不考虑）。
  - 用户账户系统（数据仅存储在本地 localStorage）。
  - 深度代码分析功能。

## 3. 技术规格

- **架构概览**：基于 Vite + React + Ant Design 构建的单页应用 (SPA)。
- **核心逻辑**：
  1. 使用 GitHub `search/repositories` API 查询符合条件的仓库（如 `created:>YYYY-MM-DD`）。
  2. 遍历结果，根据当前时间计算仓库年龄。
  3. 计算 `GrowthRate = TotalStars / DaysOld`。
  4. 按 `GrowthRate` 降序排序。
- **集成点**：GitHub REST API (v3)。
- **数据处理**：使用 Axios 进行网络请求，结果存储在 React State 中。
- **安全性与隐私**：GitHub Token 仅存储在浏览器的 `localStorage` 中，不会上传至任何服务器。

## 4. 风险与路线图

- **分阶段规划**：
  - **MVP (v1.0)**：核心查询功能、基础表格展示、Token 支持。
  - **v1.1**：增加多种排序方式（总 Stars、创建时间、增长速度切换）。
  - **v2.0**：增加趋势图预览、导出 CSV 功能。
- **技术风险**：
  - **API 限流**：无 Token 情况下 GitHub API 每分钟仅限 10 次搜索请求。
  - **计算偏差**：新创建不满 1 天的仓库，年龄计算需进行平滑处理（如取 `max(1, days)`）。
